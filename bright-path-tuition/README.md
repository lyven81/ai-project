# Bright Path Tuition Centre — AI Management Assistant

> Know every child. Find every gap. Bridge it.
> Built for Malaysia's post-UPSR PBD + UASA era.

An AI-powered management dashboard for a **premium small-class tuition centre**. The manager asks questions in plain English ("which Year 4 children are behind in Math?", "who is at risk of entering Form 1 unprepared?") and the AI agent translates each question into SQL, runs it against a 7-table SQLite database, and returns an answer — including 2-line gap-and-fix recommendations for at-risk children.

## What makes it different

- **Built for the new policy reality.** Malaysia abolished UPSR and PT3 permanently. Assessment is now Classroom-Based (PBD) + UASA, with Year 4 as the MOE anchor year from 2026. This app is built around that world — no exam countdown, no drill-to-pass.
- **Premium small-class positioning.** Max 10 children per class, 1 specialist per ~9 children. The app enforces this as a quality promise, not a constraint.
- **Year-level philosophy baked into the schema.**
  - Preschool → foundation
  - P1–P3 → build core habits
  - **P4 → Year 4 mastery anchor (MOE focus)**
  - **P5 → secondary prep Phase 1 (build forward, not backward)**
  - **P6 → secondary prep Phase 2 (ensure every child is Form 1 ready)**
- **P5/P6 gaps trigger CRITICAL escalations.** Unlike P1–P4 gaps (routine), a P5 or P6 child with an unresolved Year 4 gap is flagged as critical because the runway to secondary school is already short.
- **Gap → 2-line fix format.** The AI agent doesn't just surface problems — it proposes specific interventions with specific specialist names.

## Tech stack

| Layer | Tool |
|---|---|
| LLM | Google Gemini 2.0 Flash (via `google-generativeai`) |
| Backend | FastAPI |
| Database | SQLite (dev) → AlloyDB / PostgreSQL (prod) |
| Frontend | Vanilla HTML + CSS (served by FastAPI) |
| Container | Dockerfile → Google Cloud Run |

## Database (7 tables)

| Table | Rows | Purpose |
|---|---|---|
| `students` | 185 | Children + Year 4 mastery baselines for P5/P6 |
| `tutors` | 22 | Specialist tutors, grouped by band (preschool / P1–P3 / P4–P6) |
| `classes` | 63 | 3 preschool + 30 P1–P3 subject-specific + 30 P4–P6 subject-specific |
| `enrolments` | 553 | Which child is in which class |
| `attendance` | 3,318 | 6 weeks of weekly class sessions |
| `assessments` | 1,423 | Monthly tests + school PBD captures |
| `development_notes` | 370 | Holistic PBD dimensions (academic_mastery, critical_thinking, participation, discipline, collaboration, creativity) |

## Seeded demo stories

The database contains 9 hard-coded "stories" so the demo is never empty:

1. **Year 4 anchor gaps** — 5 P4 children flagged with core-subject mastery gaps
2. **P5 CRITICAL escalation** — 2 P5 children who entered P5 with unresolved Year 4 gaps
3. **P6 URGENT secondary risk** — 1 P6 child with a Year 4 Science gap, 5 months to Form 1
4. **P6 secondary-ready celebration** — 4 P6 children fully ready across all 5 subjects
5. **Top improvers** — 5 rising stars who jumped 15+ marks month-over-month
6. **Underperforming specialist** — 1 P4–P6 English tutor whose class improved less than her peer
7. **PBD divergence** — 3 children whose centre tests are strong but school PBD is weak
8. **Holistic development concerns** — specific children flagged on critical thinking / academic mastery dimensions
9. **Class fill-rate variance** — natural mix of at-cap, mid, and underfilled classes

## Quick start

### 1. Install dependencies
```bash
pip install -r requirements.txt
```

### 2. Build the database
```bash
python seed_generator.py
```
This creates `data.db` with all 7 tables and the 9 seeded stories.

### 3. Set your Gemini API key
```bash
cp .env.example .env
# Edit .env and paste your key from https://aistudio.google.com/apikey
```

### 4. Run locally
```bash
python main.py
```
Open http://localhost:8080 in your browser.

### 5. Try these questions in the chat
- *Which Year 4 children are struggling with core subject mastery?*
- *Show me P5 children whose current Math score is below their Year 4 baseline.*
- *Which P6 children are at risk of entering Form 1 unprepared in Science?*
- *Compare Year 4 Math mastery between Teacher Daniel and Teacher Alicia.*
- *Which classes have space for 2 or 3 more children?*
- *Who are this month's top improvers across the centre?*

## Policy context

This app is built around the Ministry of Education's permanent abolishment of UPSR and PT3. MOE has confirmed that assessment is now based on Classroom Assessment (PBD) and Ujian Akhir Sesi Akademik (UASA), with new Year 4 assessment measures being implemented in 2026 focusing on core subject mastery. Every design decision in this app — from the Year 4 anchor emphasis to the P5/P6 "build forward, not backward" rule to the 6-dimension development_notes table — reflects that policy reality.
