📋 Project Overview & Problem Statement

Challenge: Social media managers and digital marketers routinely allocate budget across multiple channels — Email, Social, Paid Search, Display, and Affiliate — based on intuition, industry benchmarks, or whatever worked last quarter. This approach makes it hard to know with confidence which channel is generating real ROI and which is silently draining budget.

Solution: Social Media Marketer runs a competitive multi-agent simulation where five AI agents — each managing one marketing channel — compete with the same starting budget over four rounds. Using a real campaign dataset, each agent picks campaigns, allocates spend, and gets scored against an ROI benchmark. The result is a ranked leaderboard plus clear recommendations: which channel to scale, which to hold, and which to cut.

Key Benefits

  • Data-backed channel decisions: Replace gut feel with simulation results grounded in real campaign uplift data
  • Same-model, different personas: All five agents use one LLM — differentiated only by system prompt personas — demonstrating how prompt design shapes AI behaviour
  • Live progress UI: Watch each agent make decisions in real time with profit/loss feedback per round
  • Actionable output: CMO-level recommendations — Scale Up, Hold, or Cut — with ROI justification
  • Replayable simulation: Each run uses randomised campaign draws, so results vary and patterns emerge across multiple runs

🖥️ Application Features

📈 Channel Baseline Table

Before the simulation starts, a baseline table shows each channel's historical average uplift from the dataset — colour-coded green or red against the benchmark. Gives context before any agent runs.

🤖 5 Competing Agents

Emma (Email), Sam (Social), Parker (Paid Search), Diana (Display), and Alex (Affiliate) each receive campaign options from their channel, decide which to run, and allocate budget. Each has a distinct strategic persona.

⚡ Live Round-by-Round Updates

A progress bar advances as each agent completes their round. Agent decision cards appear one by one showing campaign picked, uplift achieved, budget change, and one-sentence reasoning from the AI.

🏆 Final Leaderboard

After all rounds, agents are ranked by final budget. The table shows total ROI, average uplift, how many rounds hit the benchmark, and a status flag — OK or FLAGGED — for agents that missed benchmark twice consecutively.

📊 ROI Bar Chart

A horizontal bar chart visualises each channel's total ROI — green for profit, red for loss — making the performance spread immediately visible without reading the table.

💡 CMO Recommendations

Three recommendation cards — Scale Up, Hold, Cut / Review — assign every channel to an action category based on ROI and flagging status. One-click download exports the full report as Markdown.

🤖 AI Integration & Intelligence

🧠 One Model, Five Personas

All five agents use the same Claude Haiku model. Each gets a unique system prompt defining their channel focus, decision style, and strategic strength. This isolates the effect of persona on campaign selection — not model capability.

📋 Structured JSON Decisions

Each agent responds in strict JSON: campaign choice (1–3), budget allocation percentage (15–25%), and one-sentence reasoning. Structured output ensures reliable parsing and consistent simulation mechanics.

⚖️ Economic Scoring

Decisions are scored against the expected uplift from the campaign dataset. Above benchmark: the agent earns back more than it spent. Below benchmark: partial loss. Two consecutive losses trigger a FLAGGED status — mirroring real-world performance review logic.

🛡️ Fault-Tolerant Fallback

If the API returns malformed JSON, the agent falls back to a safe default choice without crashing the simulation. Ensures a complete run even under network or rate-limit conditions.

🛠️ Technical Architecture & Implementation

Frontend Stack

Streamlit Plotly Pandas Rich (CLI fallback)

Backend Stack

Python 3 Anthropic SDK Claude Haiku python-dotenv

Data

campaigns.csv 50 real campaigns 5 channels 4 objectives 5 segments

System Architecture

  • Modular Python files: agents.py, simulation.py, data.py, report.py, config.py — each file has one job
  • Session state: Streamlit session state preserves simulation results across reruns — agents and round logs are stored in st.session_state
  • Config override at runtime: Sidebar sliders modify config.py values before agents are initialised, so each run uses the user's chosen settings
  • CSV-driven scoring: Campaign uplift values come directly from the dataset — no hardcoded scores — keeping results grounded in real data
  • Exportable report: simulation_report.md is written after each run and offered as a download via Streamlit's download button

📖 Development Setup & Installation Guide

Prerequisites

  • Python 3.9+
  • Anthropic API Key from console.anthropic.com

Quick Start

# Clone the repository git clone https://github.com/lyven81/ai-project.git cd ai-project/projects/social-media-marketer # Install dependencies pip install -r requirements.txt # Add your API key # Open .env and replace "your-api-key-here" with your Anthropic API key # Launch the web UI streamlit run app.py # Or double-click run.bat (Windows)

Environment Configuration

ANTHROPIC_API_KEY=your-api-key-here

Adjustable Settings (config.py)

  • STARTING_BUDGET — Starting budget per agent (default: $1,000)
  • ROI_BENCHMARK — Minimum uplift to pass (default: 0.09 = 9%)
  • ROUNDS — Number of simulation rounds (default: 4)
  • KILL_THRESHOLD — Consecutive misses before flagging (default: 2)
  • MODEL — Claude model to use (default: claude-haiku-4-5-20251001)

🚀 Running the Simulation

# Web UI (recommended) streamlit run app.py # CLI fallback python main.py

Output Files

  • Live leaderboard and recommendations displayed in the browser after each run
  • simulation_report.md — Full round-by-round breakdown saved automatically after each run
