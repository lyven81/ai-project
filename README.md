### 1. Service Agents (7 projects)

**Who it's for:** Solo operators and micro-SMEs running their business on WhatsApp — plumbers, gardeners, car workshops, aircon technicians, court bookers, car rental, online florists.

#### Lucy — Bloom House Concierge
- **What it does:** Florist concierge that recommends, escalates, or politely declines, on the shop's website.
- **How it's built:** Vanilla HTML/CSS/JS with a real SQLite catalogue queried in-browser via sql.js, hosted on GitHub Pages.
- **Problem solved:** An online florist's one or two person team answers the same delivery and care questions by hand while sensitive cases (a late condolence stand) wait in the same queue. Lucy recommends real bouquets, answers routine questions, hands money and order decisions to a human, and refuses private or off-topic questions outright.
- **Demo:** [Try it](https://lyven81.github.io/ai-project/projects/bloom-house-concierge/index.html)
- **Deployed:** 2026-06-04

#### Plumber WhatsApp Agent
- **What it does:** AI plumber that books jobs via WhatsApp in bahasa santai. 5-step guided flow, fixed pricing.
- **How it's built:** Vanilla HTML/CSS/JS + Google Gemini 2.0 Flash with bring-your-own-key, hosted on GitHub Pages.
- **Problem solved:** Solo plumbers in Malaysia miss WhatsApp bookings while on-site and lose 2–3 jobs a week. This agent answers instantly in santai BM, scopes the problem, quotes a fixed price, books an available slot, and confirms — 24/7 with zero missed leads.
- **Demo:** [Try it](https://lyven81.github.io/ai-project/projects/plumber-whatsapp-agent/demo.html)
- **Deployed:** 2026-04-17

#### Kereta Sewa Jalan-jalan
- **What it does:** AI car rental agent for a 56-vehicle Klang Valley fleet. Books, prices, and tracks cars in plain English.
- **How it's built:** Python + FastAPI + Google Gemini 2.5 Flash with TinyDB, using a code-as-action agentic pattern.
- **Problem solved:** Small Malaysian car rental operators juggle phone calls, WhatsApp, and spreadsheets to track a 50+ vehicle fleet, and bookings slip through the cracks. This agent handles the full workflow 24/7 — checking availability, quoting price, collecting details, confirming — all in natural language.
- **Demo:** [Try it](https://lyven81.github.io/ai-project/projects/car-rental-booking-agent/demo.html)
- **Deployed:** 2026-04-08

#### Badminton Court Booking Agent
- **What it does:** Book courts with smart AI pricing and instant conflict checks. No admin work, runs 24/7.
- **How it's built:** Python + FastAPI + Google Gemini 2.5 Flash with TinyDB, using code-as-action pattern so the LLM writes Python on the fly per request.
- **Problem solved:** Court owners need a booking system that handles dynamic pricing (peak/off-peak × weekday/weekend), multi-hour bookings, and complex Malaysian-IC validation — but no off-the-shelf SaaS fits. This agent generates the booking logic dynamically for each conversation.
- **Demo:** [Try it](https://lyven81.github.io/ai-project/projects/badminton-booking-agent/demo.html)
- **Deployed:** 2025-11-24

#### Soon Heng Air-Cond Service
- **What it does:** WhatsApp customer service agent for an aircon business in Seremban & Nilai. Handles inquiry-to-follow-up in 6 stages.
- **How it's built:** Vanilla HTML/CSS/JS with a scripted 6-stage demo workflow — no backend, runs anywhere.
- **Problem solved:** Small aircon service operators lose bookings to faster-responding competitors because they can't reply while on a ladder. This agent answers in seconds, scopes the job, books the slot, sends reminders, and follows up after the job to earn Google reviews — all without the owner typing.
- **Demo:** [Try it](https://lyven81.github.io/ai-project/projects/soon-heng-aircon-agent/demo.html)
- **Deployed:** 2026-04-08

#### Hire Gardener
- **What it does:** AI agent that finds, negotiates with, and hires a grass cutting service entirely via WhatsApp.
- **How it's built:** Python + Streamlit + Ollama with Llama3 for local mock mode, plus Meta WhatsApp Cloud API for production.
- **Problem solved:** Hiring a grass cutter in Malaysia means hours of back-and-forth WhatsApp messages with multiple vendors. This agent runs the full 6-stage workflow on the user's behalf — contacting vendors, collecting quotes, booking, and verifying job completion — while vendors talk naturally to what they think is a human.
- **Demo:** [Try it](https://lyven81.github.io/ai-project/projects/hire-gardener/demo.html)
- **Deployed:** 2026-03-17

#### Wee Auto Car Care
- **What it does:** Live bay and mechanic dashboard for independent car workshops. Flags "free bay but no qualified mechanic" traps.
- **How it's built:** Vanilla HTML/CSS/JS with localStorage state and a Python seed generator — a single self-contained HTML file.
- **Problem solved:** Independent car workshops run on whiteboards and mental models, so they give wrong "come down, we're free" answers when the bay is empty but no mechanic with the right skill is available. This dashboard surfaces the 4-state booking reality (BOOKABLE / IDLE / BUSY / MULTI), preventing lost walk-in revenue.

- **How it's built:** Python + Google ADK (Agent Development Kit) + A2A protocol + Gemini 2.5 Flash + FastAPI, with pandas tool for exact metric computation.
- **Problem solved:** SME bookstore owners have months of sales data but no analyst to turn it into stocking decisions. This 4-agent system (researcher → judge → builder) returns a 600-word actionable brief — Pareto analysis, dead stock, seasonal pre-stocking — in under 3 minutes.

This 7-seat boardroom runs a weekly review where 6 AI managers (engineering, research, marketing, editor, coordinator, mentor) deliver reports on inventory, leads, content, and strategy, all grounded in a living agency state file.

### 3. Conversational Data Analytics (9 projects)

**Who it's for:** SME owners who have years of data (POS, orders, distribution) but no analyst to ask questions in plain language.

#### Delivery Delay Predictor
- **What it does:** Predicts which e-commerce orders will be delivered late before they ship, and explains why.
- **How it's built:** An XGBoost and SHAP model trained on BigQuery order data and served inside a FastAPI app on Cloud Run, with Gemini 2.5 Flash on Vertex AI for plain-language recommendations and a live Chart.js dashboard.
- **Problem solved:** E-commerce fulfillment teams usually discover a late delivery only after the customer complains or asks for a refund. This scores every open order's delay risk before dispatch, names the top driver behind each at-risk order, and recommends what to do, so teams act before the delivery fails.
- **Demo:** [Try it](https://delivery-delay-predictor-522143897885.asia-southeast1.run.app)
- **Deployed:** 2026-07-07

#### Voucher Analyzer
- **What it does:** Shows online sellers which vouchers actually made money, not just which ones sold.
- **How it's built:** A Python and pandas incrementality engine (baseline-counterfactual) with a recovery check, served as static HTML with a keyless, client-side sql.js governed query layer.
- **Problem solved:** A small SEA e-commerce seller runs vouchers every campaign but cannot tell which drove genuinely new sales versus subsidising buyers who would have bought anyway. This separates incremental sales from baseline, scores every voucher, and turns one flat policy into a per-category plan.
- **Demo:** [Try it](https://lyven81.github.io/ai-project/projects/voucher-analyzer/demo.html)
- **Deployed:** 2026-06-17

#### Sundry Shop Assistant
- **What it does:** Voice advisor for busy kedai runcit owners. Ask POS data in Malay, get answers from 10 MCP tools.
- **How it's built:** Gemini Live API + FastAPI + WebSocket + 10 pandas-backed MCP tools on Cloud Run.

This voice advisor answers in santai BM — "best seller today," "should I reorder Milo," "member vs visitor spend" — without the owner needing to type or remember how to export reports.
- **Demo:** [Try it](https://sundry-shop-assistant-522143897885.asia-southeast1.run.app)

This live agent answers distribution questions in seconds — "which outlets are slipping," "best revenue per outlet by channel" — replacing a 2-hour spreadsheet hunt with one plain-English question.

#### Market Research Agent

This measures how occupancy responds to price by season, proves the estimate against a known answer, and gives the profit-maximizing rate per listing per month.

### Local `.env` files (do not commit)
Two projects keep a local `.env` for running their full Python server on this machine only:

| File | Variables | Used by |
|---|---|---|
| `bright-path-tuition/.env` | `GOOGLE_API_KEY`, `PORT` | local FastAPI server (`bright-path-tuition/main.py`) |
| `projects/bahai-chinese-translation-workbench/.env` | `ANTHROPIC_API_KEY` | local server (`projects/bahai-chinese-translation-workbench/app.py`) |

Findings (2026-06-24): both are **untracked** and were **never committed** to git history; neither key value appears in any tracked file (no leak, no rotation needed). The **published demo links are static GitHub Pages pages** (`demo.html`) and do **not** read these `.env` files, so the demos run regardless. These files are kept locally by choice and are not deleted.

Action for the future: they are **not** yet covered by `.gitignore`, so never run `git add -A` / `git add .` in this repo. Stage files by name. (Adding a `.env` rule to `.gitignore` is the clean fix when ready.)

### Project subheading rule (one line, <=10 words, identical everywhere)
Each project has ONE subheading: clear, direct, simple everyday language describing what the project does, **10 words maximum**. The SAME text must appear in all four places: the showcase hero (`PROJECT.tagline`), the homepage `items[].dek`, the `project-categories.html` `CATS[].projects[].desc`, and the dedicated `category-*.html` `CATEGORY.projects[].desc`. No emojis, no em-dashes, no filler. **Drop the redundant word "AI"** (the whole portfolio is AI); keep "agent"/"multi-agent" (architecture) and keep "AI" only when load-bearing (e.g. the agent is the customer in agentic commerce). Enforced by `ai-project-publish`.
