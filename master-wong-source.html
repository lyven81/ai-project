About This Code Showcase

This curated selection shows how Master Wong implements the traditional three-coin I Ching casting method digitally, then uses Claude to interpret the resulting hexagram(s) in the context of the user's specific question — all in Mandarin Chinese.

The app supports both single hexagram readings and paired (comparative) readings, with a follow-up chat for deeper exploration of the oracle's guidance.

Core: Hexagram Casting & Reading

The server handles the full I Ching flow: (1) Claude recommends a reading type based on the user's question, (2) hexagram(s) are cast using the three-coin method, (3) Claude interprets the hexagram in context. Paired readings cast two hexagrams for comparative insight.

server.mjs — Cast & Read Routes
import { config } from './iching.config.mjs' import { castHexagram } from './apis/casting.mjs' import { getRecommendation, getInitialReading, getChatReply } from './apis/reading.mjs' // Recommend reading type based on user's question app.post('/api/recommend', async (req, res) => { const { userMessage } = req.body const recommendation = await getRecommendation(userMessage.trim()) res.json(recommendation) }) // Cast hexagram(s) — single or paired reading app.post('/api/cast', async (req, res) => { const { readingType } = req.body if (readingType === 'paired') { // Two hexagrams for comparative insight const cast1 = castHexagram() const cast2 = castHexagram() res.json({ cast1, cast2, readingType }) } else { const cast1 = castHexagram() res.json({ cast1, readingType }) } }) // Generate AI interpretation of the cast hexagram(s) app.post('/api/read', async (req, res) => { const { question, cast1, cast2 } = req.body const reading = await getInitialReading(question, cast1, cast2 || null) res.json({ reading }) })

Configuration & Follow-up Chat

The config is minimal — just port, session directory, and Claude model. The chat endpoint carries the full reading context (question, cast hexagrams, initial interpretation, and conversation history) so Master Wong can provide consistent, grounded follow-up guidance.

iching.config.mjs — App Configuration
import 'dotenv/config' export const config = { port: process.env.PORT || 3002, sessionsDir: './sessions', claudeModel: 'claude-sonnet-4-6' }
server.mjs — Context-Aware Follow-up Chat
// Chat follow-up — carries full reading context app.post('/api/chat', async (req, res) => { const { question, cast1, cast2, initialReading, history, message } = req.body const reply = await getChatReply({ question: question || '', cast1, cast2: cast2 || null, initialReading: initialReading || '', history: history || [], message }) res.json({ reply }) })

Technical Implementation Notes

Key Design Decisions

  • Traditional three-coin method: The castHexagram() function simulates the classic I Ching coin-toss six times to build a hexagram from bottom to top, including changing lines
  • 64-hexagram reference: Complete hexagram data in hexagrams.mjs provides names, trigram compositions, and traditional meanings that feed into Claude's interpretation
  • Paired readings: For complex decisions, two hexagrams are cast and interpreted together — a feature that mirrors the traditional "before and after" consultation style
  • All output in Mandarin: Error messages, API responses, and the entire user experience are in Chinese (e.g., "请输入您的问题", "王师父暂时无法回应")
  • Session persistence: Each reading session is saved as a timestamped JSON file for future review and reflection
