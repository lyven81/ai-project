🔍 About This Code Showcase

These snippets show the parts that make this both a triage tool and an honest evaluator: the five-way Gemini lane call, the data prep that hides the fraud label, the deterministic gate that can overrule the model, and the imbalance-aware metrics with a Wilson interval.

The full page is a single self-contained HTML file with no backend. Real claims are sampled offline by a keyless Python script and shipped as a static JavaScript asset. Scoring runs live in the browser with the visitor's own Gemini key.

🧠 The Five-Way Lane Call (Gemini, structured output)

Unlike a yes-or-no classifier, this asks Gemini for a strict JSON object with a five-way handling lane plus a fraud risk, a confidence, and a one-line reason. A response schema with an enum forces the shape, temperature 0.2 keeps it steady, and the claim facts go in while the true outcome never does.

📄 demo.html · scoreClaimLive()
const RESP_SCHEMA = {type:"object", properties:{ lane:{type:"string", enum:["Fast-track","Standard review", "Complex / high-value","Investigate (SIU)","Repudiate-candidate"]}, fraud_risk:{type:"number"}, confidence:{type:"number"}, reason:{type:"string"} }, required:["lane","fraud_risk","confidence","reason"]}; async function scoreClaimLive(key, c, retries=4){ const payload = {system_instruction:{parts:[{text:SYSTEM_PROMPT}]}, contents:[{parts:[{text:"Classify this claim:\n"+factSheet(c)}]}], generationConfig:{responseMimeType:"application/json", responseSchema:RESP_SCHEMA, temperature:0.2}}; for(let a=0; a<=retries; a++){ const res = await fetch(GEMINI_URL+"?key="+encodeURIComponent(key), {method:"POST", ...}); if(res.ok){ const v = JSON.parse(out.candidates[0].content.parts[0].text); return {lane:LANE_MAP[v.lane.toLowerCase()], fraud_risk:clamp(+v.fraud_risk,0,1), confidence:clamp(+v.confidence,0,1), reason:v.reason}; } // free-tier keys are rate-limited; back off and retry on 429/503 if((res.status===429||res.status===503) && a<retries){ await sleep(4000*(a+1)); continue; } throw new Error("HTTP "+res.status); } }

🔒 The Honest Data Prep (label kept out of the prompt)

A keyless Python script stratifies the real dataset to 40 fraud and 110 legitimate claims, maps the bucketed columns into a fact sheet for the model, and attaches isFraud for scoring only. The label travels inside each claim but is never placed in the prompt.

🚦 The Gate That Can Overrule the Model

The model proposes a lane, but a short deterministic rule, not the LLM, makes the routing call. Any claim whose fraud risk crosses the safety dial is pulled into investigation; everything else follows the lane the model suggested. The rule is auditable, the override is explicit.

📄 demo.html · decorate(claims, threshold)
function decorate(claims, th){ return claims.map(c => { if(!c.scored) return {...c, lane:"unscored", predFraud:0}; // at or above the dial it goes to investigation; a claim the model itself // routed to SIU also counts as a fraud prediction (blueprint rule C) const lane = c.risk >= th ? "siu" : c.modelLane; const predFraud = lane === "siu" ? 1 : 0; return {...c, lane, predFraud, reason:c.modelReason}; }); }

📊 The Fraud Metrics Engine (built for a rare event)

One pass over the scored claims builds the confusion matrix, then catch rate (recall), flag accuracy (precision), false-alarm rate, and F1. On a 6% event these matter far more than raw accuracy, which is why the scorecard leads with them.

📄 demo.html · metrics(claims)
function metrics(claims){ let TP=0, FP=0, FN=0, TN=0; claims.forEach(c => { if(c.predFraud && c.isFraud) TP++; else if(c.predFraud && !c.isFraud) FP++; else if(!c.predFraud && c.isFraud) FN++; // missed fraud, the expensive failure else TN++; }); const prec = TP/(TP+FP||1); // when it flags, how often right const rec = TP/(TP+FN||1); // catch rate, the number to protect const fpr = FP/(FP+TN||1); // honest customers wrongly flagged return {TP, FP, FN, TN, prec, rec, fpr, f1:2*prec*rec/((prec+rec)||1), recCI:wilson(TP, TP+FN), precCI:wilson(TP, TP+FP)}; }

📐 The Confidence Interval (Wilson score)

Every headline number carries a 95% interval, so the audience knows the margin of error and can watch it tighten as the sample grows. This is what answers "you only scored 150, is that enough?".

📄 demo.html · wilson(k, n)
// 95% Wilson score interval for a proportion k/n function wilson(k, n, z){ z = z || 1.96; if(n===0) return [0,0]; const p=k/n, d=1+z*z/n; const c=(p+z*z/(2*n))/d; const h=(z*Math.sqrt(p*(1-p)/n + z*z/(4*n*n)))/d; return [clamp(c-h,0,1), clamp(c+h,0,1)]; }
