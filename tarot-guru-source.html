Tarot Guru

AI-powered personal tarot reading with context-aware spreads and follow-up chat

Node.js Express Claude API ES Modules

About This Code Showcase

This curated selection shows how Tarot Guru implements a multi-step reading flow: the user describes their situation, Claude recommends a spread, cards are drawn programmatically, Claude interprets the spread, and the user can ask follow-up questions.

The architecture separates card data, spread logic, and AI interpretation into distinct modules under apis/, keeping each concern clean and testable.

Core: Multi-Step Reading Flow

The server orchestrates a four-step flow: (1) user describes their question, (2) Claude recommends a spread type, (3) cards are drawn randomly from the 78-card deck, (4) Claude interprets the drawn cards in context. A fifth endpoint handles follow-up chat.

server.mjs — Recommend, Draw & Read Routes
import { getRecommendation, getInitialReading, getChatReply } from './apis/reading.mjs' import { drawSpread } from './apis/spreads.mjs' // Step 1: Claude reads the user's context and recommends a spread app.post('/api/recommend', async (req, res) => { const { userMessage } = req.body const result = await getRecommendation(userMessage) res.json({ ok: true, ...result }) }) // Step 2: Draw cards for the recommended spread type app.post('/api/draw', async (req, res) => { const { spreadType } = req.body const drawnCards = drawSpread(spreadType) res.json({ ok: true, drawnCards }) }) // Step 3: Claude interprets the drawn cards app.post('/api/read', async (req, res) => { const { question, drawnCards } = req.body const reading = await getInitialReading(question, drawnCards) res.json({ ok: true, reading }) }) // Step 4: Follow-up conversation with full reading context app.post('/api/chat', async (req, res) => { const { question, drawnCards, initialReading, history, message } = req.body const reply = await getChatReply({ question, drawnCards, initialReading, history, message }) res.json({ ok: true, reply }) })

Session Persistence

Each completed reading can be saved to a timestamped JSON file. This lets users revisit past readings and tracks the full context — question, drawn cards, AI interpretation, and follow-up chat history.

server.mjs — Save Session Route
// POST /api/save — Save the complete reading session to disk app.post('/api/save', async (req, res) => { const { session } = req.body const now = new Date() const pad = n => String(n).padStart(2, '0') // Filename format: 2026-03-28-14-30.json const filename = `${now.getFullYear()}-${pad(now.getMonth()+1)}-${pad(now.getDate())}-${pad(now.getHours())}-${pad(now.getMinutes())}.json` const filepath = join(config.sessionsDir, filename) writeFileSync(filepath, JSON.stringify({ savedAt: now.toISOString(), ...session }, null, 2)) res.json({ ok: true, filename }) })

Technical Implementation Notes

Key Design Decisions

  • Context-aware spread selection: Claude analyzes the user's question to recommend the most appropriate tarot spread (e.g., Celtic Cross for complex life questions, Three-Card for quick guidance)
  • Programmatic card drawing: Cards are drawn randomly from the full 78-card Rider-Waite deck via spreads.mjs, keeping the randomness separate from AI interpretation
  • Stateful follow-up chat: The chat endpoint receives the original question, drawn cards, initial reading, and full conversation history — so Claude can reference specific cards and prior insights
  • Session saving: Complete reading sessions are saved as JSON files with timestamps, enabling users to review and reflect on past readings
  • Minimal dependencies: Only Express and dotenv — the entire app runs with two npm packages
