📋 Project Overview & Problem Statement

Challenge: Motor claims arrive in a flood, and most are honest. A handler has to do two jobs at once: send each claim to the right desk, and catch the rare fraud hiding in the pile. Rush and you pay the fraud; over-investigate and honest customers wait weeks. And when the board asks how good the fraud triage really is, there is no measured answer, only a vendor's black-box score that was never tested on this insurer's own claims.

Solution: This tool runs both jobs on one screen and stays honest about which one it can actually prove. It sorts every claim into one of five handling lanes for operational routing, and on the single axis that carries a real label, fraud, it scores its own flags live against the recorded outcome, with a 95% confidence interval on every figure.

Key Benefits

  • Two jobs, one screen: operational lane-sorting and a measured fraud check, kept separate so neither oversells the other
  • Honest scope: fraud is graded against the real label; the four routing lanes are shown as a distribution and never dressed up as accuracy
  • Tuned for a rare event: with fraud near 6 in 100, the scorecard leads on catch rate and false alarms, the numbers an SIU team actually argues over, not a flattering accuracy figure
  • One dial for the real trade-off: catch more fraud and bother more honest customers, or the reverse, recomputed live as you drag it
  • Governance built in: no claim is auto-denied, a repudiation candidate always reaches a person, and every flag shows the facts behind it

🖥️ Application Features

🗂️ Five-Lane Triage Console

The operational half. Every claim is sorted into fast-track, standard review, complex or high-value, investigate (SIU), or repudiate-candidate, with a headline percentage cleared and a tap-to-filter view of each lane. This is routing, shown as a distribution, not graded against truth.

✅ Fraud QC Scorecard

The measured half. Catch rate, flag accuracy, F1, false-alarm rate, and a four-box confusion matrix, all checked against the real FraudFound_P label, each carrying a 95% confidence interval. Plain-English first, the technical figure underneath, nothing hidden.

🎚️ Safety Dial

A fraud-risk threshold slider that re-splits the investigate and clear piles live and redraws a catch-rate-versus-false-alarm curve. Three presets, catch more fraud, balanced, or fewer false alarms, make the trade-off concrete for an SIU lead.

📐 Calibration and Segments

A stated-risk-versus-actual-fraud chart that exposes whether the model's certainty holds up on a rare event, plus a fraud-rate-by-accident-area sense check. The honesty layer that stops a high accuracy number from telling a comforting lie.

📄 Check One Claim

Pick any single claim from a scrollable list, or draw one at random, and see exactly how it is handled: its lane, fraud risk, the plain-English reason, and the clear-versus-human outcome. Then reveal whether it was really fraud. A worked example, deliberately kept separate from the measured scorecard, which needs a batch.

🤖 AI Integration & Intelligence

🧠 Five-Way Lane Call (Gemini AI)

This is not a yes-or-no verdict. From the claim facts alone, Gemini returns a five-way handling lane plus a fraud risk, a confidence, and a one-line reason, as structured JSON. The prompt never sees the true outcome, so the later scoring stays honest.

🎯 Only the Fraud Call Is Graded

The design choice that keeps the tool honest: of the five lanes, only fraud carries a real label, so only the fraud flags are scored. The routing lanes are reported as a distribution. The page says so out loud rather than implying all five are accurate.

🚦 The Gate Can Overrule the Model

A short deterministic rule, not the LLM, makes the routing call. It pulls any claim into investigation once its fraud risk crosses the dial, and it never lets a repudiation candidate skip a human. The model proposes; the auditable rule disposes.

📊 Calibration Under Imbalance

When fraud is 6 in 100, a model can look 94 percent accurate by clearing everything. So the tool watches whether high stated risk actually lands on fraud more often, and rebalances the demo sample to roughly 27 percent fraud, disclosed on the page, so recall is measurable at all.

📦 The QC Pack: From Tool to Operating Procedure

A demo proves the model works; running it inside a claims team needs the operating layer around it. The companion QC Pack ships four artifacts that turn the evaluator into a repeatable workflow an adjuster or an offshore handler can follow.

📋 Claims Classification Guide

One entry per lane: definition, when to route there, when to escalate, edge cases, and a worked example. What it is for: giving every handler the same written definition of each lane, so triage stays consistent from desk to desk.

🧠 Prompt Library

The primary classifier prompt, plus a tie-breaker for borderline lanes and a re-check before high-stakes flags. What it is for: reproducing and tuning the model's behaviour deliberately, instead of guessing at prompt changes.

🗂️ Good / Bad / Gray Training Set

Eight worked example claims with the correct lane and the reason, marked clear, ambiguous, or tempting-wrong. What it is for: onboarding a new adjuster fast and calibrating judgement on the cases that actually trip people up.

✅ QC SOP

The daily review-queue steps and the weekly scorecard the team maintains. What it is for: running the queue day to day, and keeping the model honest over time through a standing trust contract.

🛠️ Technical Architecture & Implementation

Frontend Stack

Single-file HTML Vanilla JavaScript Inline SVG charts No build step

AI & Data

Google Gemini (gemini-2.5-flash) Structured JSON output BYO-key, client-side Python + pandas (data prep)

Deployment & Infrastructure

GitHub Pages Static asset (claims.js) No backend

System Architecture

  • No backend, no server cost: the page calls Gemini directly from the browser with the visitor's own key
  • Real data as a static asset: a keyless Python script samples the 15,420-row Kaggle dataset and emits 150 stratified real claims as data/claims.js, loaded via <script src> so it works on GitHub Pages and offline
  • Honest scoring: the true fraud label travels with each claim but is used only after the model answers, never shown to it
  • Typed verdict to UI: Gemini's lane and fraud risk drive the confusion matrix, the metrics, the lane distribution, and the tradeoff chart
  • Self-contained: classification, scoring, charts, and the safety dial all run in one HTML file

🔁 Potential Use Cases

Strip away the motor-insurance specifics and the pattern reusable here is the triage-plus-audit loop: sort each record into lanes, let a model rate the rare flag, clear the easy ones, escalate the rest, and grade only the call you have a real answer key for. It fits wherever two things hold:

  • A high-volume queue with a rare flag: most records are routine, a small fraction need scrutiny, and the cost of missing the flag is high.
  • An answer key for that one flag: past records eventually record whether the flag was real, so the model's flags can be scored even when the routing lanes cannot.

Two examples that fit, each with the catch that decides how far to trust the scorecard:

🏥 Medical Billing Review

Classify each submitted claim line as anomalous or routine, with a confidence score. Auto-approve the clean lines, route the suspicious ones to a reviewer, and score the model against which lines were later recovered or written off.

Condition to reuse the framework: you hold historical adjudication outcomes (paid, recovered, denied). The catch is that the answer key is partial for lines no one ever audited, so use it as decision support, not an automatic denial.

📦 Warranty Claim Triage

Classify each warranty claim as likely valid or likely abuse, with a confidence score. Fast-track the clear ones, escalate the doubtful ones, and score against which claims were ultimately honoured or rejected.

Condition to reuse the framework: you define a concrete outcome up front, for example honoured at final assessment. The label is only as honest as that definition, so it must be stated plainly.

The shorter the gap between a flag and its confirmed outcome, the more the scorecard is worth. Motor fraud sits near the good end: the label is recorded for every claim, even though it is rare. Medical billing and warranty triage are usable too, as long as the page is upfront about where their answer keys are partial or slow.
