# Hoo Tim Analytics Agent

Live AI agent + MCP-style tools for a fictional Selangor snack distributor. Single-page BYO-Gemini-key demo hosted on GitHub Pages.

**Live demo:** https://lyven81.github.io/ai-project/projects/hoo-tim-analytics-agent/demo.html

**Project outline:** https://lyven81.github.io/ai-project/hoo-tim-analytics-agent.html

## Files

| File | Purpose |
|---|---|
| `demo.html` | The interactive agent UI — two-panel brown/gold layout, API key input, 15 preset questions |
| `data.js` | Pre-aggregated distribution data (129 KB) — emitted by `precompute.py` |
| `precompute.py` | Reads `dataset.csv`, builds 11 aggregated views, writes `data.js` |
| `dataset.csv` | Source: 6,074 order lines, December 2025, generated by `../hoo tim biscuits/generate_dataset.py` |

## MCP tools (5)

- `get_aggregate(dim)` — district / channel / category / sku roll-up
- `get_top_n(dim, n, direction)` — top or bottom N by metric
- `get_cross_tab(row_dim, col_dim)` — 2D breakdowns
- `get_delivery_performance(group_by)` — on-time / slightly late / very late
- `get_time_series(period)` — daily trend or Christmas–New Year campaign window

## Architecture

1. User enters Gemini API key (kept in browser session only)
2. User picks a question (or types one)
3. JS keyword dispatcher picks relevant MCP tools and runs them against `data.js`
4. Tool outputs + question are sent to Gemini 2.0 Flash with strict grounding rules
5. Markdown answer renders with table + insight + recommendation
