1. Schema Intelligence & Auto-Detection Engine

The foundation of the system is its ability to understand your data automatically. This function analyzes the dataset and creates a comprehensive schema description that helps the AI generate accurate code. 

def build_schema_block(df: pd.DataFrame) -> str:
    """
    Build a comprehensive schema description for the AI agent.
    Includes column names, types, unique values, and sample data.

    This function intelligently analyzes your dataset structure:
    - For categorical columns: Shows value distribution
    - For numeric columns: Shows min/max/mean statistics
    - For high-cardinality columns: Shows sample values
    """
    schema_lines = []
    schema_lines.append("DATASET SCHEMA:")
    schema_lines.append(f"Total Rows: {len(df):,}")
    schema_lines.append(f"Total Columns: {len(df.columns)}")
    schema_lines.append("\nCOLUMNS:")

    for col in df.columns:
        dtype = str(df[col].dtype)
        unique_count = df[col].nunique()
        null_count = df[col].isnull().sum()

        schema_lines.append(f"\n- {col}:")
        schema_lines.append(f"  Type: {dtype}")
        schema_lines.append(f"  Unique values: {unique_count:,}")
        schema_lines.append(f"  Null values: {null_count:,}")

        # Show sample values for categorical/low-cardinality columns
        if unique_count <= 20:
            samples = df[col].value_counts().head(10).to_dict()
            schema_lines.append(f"  Sample values: {samples}")
        else:
            # For high-cardinality columns, show range or examples
            if dtype in ['int64', 'float64']:
                min_val = df[col].min()
                max_val = df[col].max()
                mean_val = df[col].mean()
                schema_lines.append(f"  Range: {min_val:.2f} to {max_val:.2f} (mean: {mean_val:.2f})")
            else:
                samples = df[col].dropna().sample(min(5, len(df[col].dropna()))).tolist()
                schema_lines.append(f"  Sample values: {samples}")

    return "\n".join(schema_lines)

Key Features:

  • Automatic Type Detection: Identifies numeric vs categorical columns
  • Statistical Analysis: Computes min/max/mean for numeric data
  • Value Distribution: Shows frequency counts for categorical data
  • Missing Data Detection: Tracks null values per column
  • Smart Sampling: Provides relevant examples based on data type

2. Natural Language to Code Generation Engine

This is where the magic happens. The AI agent receives your question and the dataset schema, then generates executable Python code that answers your question precisely. 

def generate_llm_code(
    query: str,
    schema: str,
    num_rows: int,
    model_name: str = "gemini-2.0-flash-exp"
) -> str:
    """
    Generate executable Python code using Gemini LLM.

    Example Transformation:
    Input:  "Which product category is making loss because of discount?"
    Output: Python code that:
            1. Filters orders with discounts
            2. Groups by category
            3. Calculates profit margins
            4. Identifies loss-making categories
            5. Generates visualization

    Args:
        query: User's filtering/analysis question
        schema: Dataset schema information (from build_schema_block)
        num_rows: Number of rows in dataset
        model_name: Gemini model to use

    Returns:
        Generated Python code as string
    """
    # Format the prompt with user query and schema
    formatted_prompt = PROMPT.format(
        schema=schema,
        query=query,
        num_rows=num_rows
    )

    print(f"ü§ñ Generating code for query: '{query}'")
    print(f"üìè Prompt length: {len(formatted_prompt):,} characters")
    print(f"üîÑ Calling Gemini model: {model_name}...")

    try:
        # Initialize the Gemini model
        model = genai.GenerativeModel(model_name)

        # Generate response
        response = model.generate_content(formatted_prompt)

        # Check for blocking or empty response
        if hasattr(response, 'prompt_feedback') and response.prompt_feedback.block_reason:
            block_reason = response.prompt_feedback.block_reason
            print(f"üö´ Response blocked: {block_reason}")
            return f'STATUS = "error"\nanswer_text = "Response blocked: {block_reason}"'

        if not response.text or response.text.strip() == "":
            print("‚ö†Ô∏è Empty response from LLM")
            return 'STATUS = "error"\nanswer_text = "Empty response. Please rephrase."'

        generated_code = response.text.strip()
        print(f"‚úÖ Code generated successfully ({len(generated_code)} characters)")

        return generated_code

    except Exception as e:
        print(f"‚ùå Error calling Gemini API: {str(e)}")
        return f'STATUS = "error"\nanswer_text = "API Error: {str(e)}"'

Code Generation Capabilities:

  • Multi-Pattern Recognition: Handles filtering, aggregation, grouping, visualization
  • Context-Aware: Uses schema information to generate accurate column references
  • Error Resilient: Includes try-except blocks and status variables
  • Safety Filters: Checks for blocked responses and empty outputs
  • Transparent Logging: Prints progress updates for user visibility

3. Safe Code Execution Environment

Security is paramount. This function executes the AI-generated code in a controlled, sandboxed environment that prevents harmful operations while still allowing powerful data analysis. 

def execute_generated_code(code_text: str, df: pd.DataFrame) -> Dict[str, Any]:
    """
    Execute generated Python code in a safe namespace.

    Security Features:
    1. Works on DataFrame COPY (original data never modified)
    2. Limited namespace (only pandas, numpy, matplotlib, seaborn)
    3. No file system access
    4. No network operations
    5. Comprehensive error handling

    Args:
        code_text: Python code to execute (may include markdown)
        df: DataFrame to make available in execution context

    Returns:
        Dictionary with STATUS and answer_text
    """
    # Extract executable code (removes markdown if present)
    code = _extract_execute_block(code_text)

    if not code.strip():
        return {
            "STATUS": "error",
            "answer_text": "No executable code found in LLM response"
        }

    print("\n" + "="*60)
    print("EXECUTING GENERATED CODE:")
    print("="*60)
    print(code)
    print("="*60 + "\n")

    # Create safe execution namespace (sandboxed environment)
    namespace = {
        "df": df.copy(),  # Work on a copy to avoid modifying original
        "pd": pd,
        "np": np,
        "plt": plt,
        "sns": sns,
        "STATUS": "unknown",
        "answer_text": "No answer generated"
    }

    try:
        # Execute the code in sandboxed namespace
        exec(code, namespace)

        # Extract results from namespace
        status = namespace.get("STATUS", "unknown")
        answer = namespace.get("answer_text", "No answer provided")

        return {
            "STATUS": status,
            "answer_text": answer
        }

    except Exception as e:
        error_msg = f"Execution error: {str(e)}"
        print(f"‚ùå {error_msg}")
        return {
            "STATUS": "error",
            "answer_text": error_msg
        }

Security & Safety Features:

  • DataFrame Isolation: Works on copy of data to prevent accidental modifications
  • Limited Namespace: Only exposes pandas, numpy, matplotlib, seaborn libraries
  • No File Access: Prevents reading/writing files outside the sandbox
  • No Network Access: Blocks HTTP requests and external connections
  • Error Containment: Catches all exceptions and returns structured error messages
  • Status Tracking: Every execution returns explicit success/error status
